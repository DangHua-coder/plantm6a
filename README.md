# PlantM6A

PlantM6A is a Python toolkit for m6A RNA modification analysis in plant genomes and comparative species studies. It provides command-line and Python APIs for exon-level m6A statistics, batch summaries, sequence motif analysis, and conserved m6A site analysis using orthologous genes.

> Development status: alpha. The core APIs are usable, but interfaces may still change before a stable release.

## Features

- Count exon bases, exon A bases, m6A sites, and normalized m6A/A rates from FASTA, GTF/GFF, and BED files.
- Run batch analysis across multiple species or samples from a YAML config.
- Classify m6A-centered motifs as RAC/GAT/Others or all 16 centered-A 3-mers.
- Translate CDS FASTA records to protein FASTA for orthology workflows.
- Extract pairwise ortholog tables from OrthoFinder output.
- Identify differential m6A sites between two conditions of one species.
- Identify and summarize conserved m6A sites between species pairs.
- Plot Circos-style chord diagrams from conservation summaries.

## Installation

For local development:

```bash
git clone https://github.com/yourusername/plantm6a.git
cd plantm6a
python -m pip install -e .
```

Install all optional analysis dependencies for the full toolkit:

```bash
python -m pip install -r requirements.txt
```

Or install extras by feature:

```bash
python -m pip install -e ".[motif]"
python -m pip install -e ".[conservation]"
python -m pip install -e ".[visualization]"
python -m pip install -e ".[dev]"
```

Some conservation commands also require external tools, especially `samtools` for FASTA indexing/extraction and OrthoFinder outputs for ortholog tables.

## Input Formats

- **Genome**: FASTA file. Motif and conservation analysis should use an indexed FASTA when possible (`samtools faidx genome.fa`).
- **Annotation**: GTF or GFF3. Exon features are used for statistics and motif analysis; exon and CDS features are used for conservation region labels.
- **m6A sites**: BED-like file with at least `chrom`, `start`, and `end` columns. Strand in column 6 is used when present.
- **Batch config**: YAML with a top-level `species` list. Each item must include `name`, `genome`, `annotation`, and `bed`.
- **Ortholog table**: TSV generated by `plantm6a extract-orthologs`, with `OG_ID`, species gene columns, and `type`.

## Command-Line Usage

Show all commands:

```bash
plantm6a --help
```

### Single-sample statistics

```bash
plantm6a stats \
  --genome ../test/TAIR10.fa \
  --gtf ../test/TAIR10.release55.gtf \
  --bed ../test/Col_seedling7days_mR_coverageover20_ratioover0.05.bed6 \
  --output arabidopsis_stats.tsv
```

### Batch statistics

```bash
plantm6a batch \
  --config examples/species_config.example.yaml \
  --output m6a_batch_summary.tsv
```

### Motif analysis

```bash
plantm6a motif \
  --genome ../test/TAIR10.fa \
  --annotation ../test/TAIR10.release55.gtf \
  --bed ../test/Col_seedling7days_mR_coverageover20_ratioover0.05.bed6 \
  --mode simple \
  --output arabidopsis_motif.tsv
```

Use `--mode complete` to report all 16 centered-A 3-mers.

### Per-site annotation

Use `annotate-sites` when you need one row per BED site with region, motif, gene, strand, and ratio annotations:

```bash
plantm6a annotate-sites \
  --genome ../test/TAIR10.fa \
  --annotation ../test/TAIR10.release55.gtf \
  --bed ../test/Col_seedling7days_mR_coverageover20_ratioover0.05.bed6 \
  --output arabidopsis_sites_annotated.tsv
```

The output columns are `site_index`, `chrom`, `start`, `end`, `position`, `bed_name`, `ratio`, `bed_strand`, `assigned_strand`, `motif`, `motif_type`, `sequence_context`, `region`, `gene_id`, `gene_name`, `transcript_id`, `transcript_position`, `overlap_count`, and `annotation_status`.

Region labels are `5utr`, `cds`, `3utr`, `intron`, `noncoding`, and `intergenic`. The `ratio` column is read from BED column 5. By default, overlapping transcripts are resolved to one best annotation per BED site; use `--overlap-policy all` to emit all transcript overlaps.

### Differential m6A

Compare two BED6 m6A ratio files from the same species and write site-level, region-level, gene-level, GO-input, and figure outputs:

```bash
plantm6a diff-m6a \
  --wt Col_seedling7days_ratioover0.1.bed6 \
  --mut EJC_C11_seedling7days_ratioover0.1.bed6 \
  --annotation ../test/TAIR10.release55.gtf \
  --output results/Col_vs_C11 \
  --wt-name Col \
  --mut-name C11
```

Sites are matched by exact `chrom`, BED `start`, and strand. BED column 5 is interpreted as the m6A ratio. Main outputs are `diff_m6a_all.tsv`, `diff_m6a_by_region.tsv`, `diff_m6a_by_gene.tsv`, `diff_m6a_summary.tsv`, `go_input/`, and `figures/`.

### Conservation workflow

Translate CDS FASTA files to proteins for OrthoFinder:

```bash
plantm6a translate-cds --input species.cds.fa --output species.protein.fa
```

After running OrthoFinder, extract pairwise ortholog tables:

```bash
plantm6a extract-orthologs \
  --orthogroups orthofinder_out/Results_*/Orthogroups/Orthogroups.tsv \
  --orthologues-dir orthofinder_out/Results_*/Orthologues \
  --output-dir pairwise_tables
```

Identify conserved m6A sites for one species pair:

```bash
plantm6a conserve-pair \
  --sp-a arabidopsis \
  --sp-b rice \
  --genome-a arabidopsis.fa \
  --genome-b rice.fa \
  --annot-a arabidopsis.gtf \
  --annot-b rice.gtf \
  --m6a-a arabidopsis_m6a.bed \
  --m6a-b rice_m6a.bed \
  --ortholog-file pairwise_tables/arabidopsis_vs_rice.tsv \
  --output conserved_results/arabidopsis_vs_rice_conserved_m6A.tsv
```

Summarize all conserved-site result files:

```bash
plantm6a summarize-conservation \
  --conserved-dir conserved_results \
  --output-dir summary
```

Plot a chord diagram from the summary output:

```bash
plantm6a plot-chord \
  --input summary/pair_region_breakdown.tsv \
  --output figures/conserved_m6A_chord \
  --min-sites 100
```

## Python API

```python
from plantm6a.analysis import analyze

result = analyze(
    genome_file="genome.fa",
    annotation_file="annotation.gtf",
    bed_file="sites.bed",
    verbose=False,
)
print(result["m6A_per_1000A"])
```

```python
from plantm6a.analysis import analyze_motifs

motifs = analyze_motifs(
    genome_path="genome.fa",
    annotation_path="annotation.gtf",
    bed_path="sites.bed",
    mode="complete",
    verbose=False,
)
print(motifs["GAT_count"])
```

```python
from plantm6a.analysis import annotate_m6a_sites

rows = annotate_m6a_sites(
    genome_path="genome.fa",
    annotation_path="annotation.gtf",
    bed_path="sites.bed6",
    output_path="sites_annotated.tsv",
    verbose=False,
)
print(rows[0]["region"], rows[0]["motif"], rows[0]["ratio"])
```

